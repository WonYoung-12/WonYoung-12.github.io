개발 환경

Android

Pet Advisor는 Android App입니다. RecyclerView, CardView 등의 기본적인 내용부터 Retrofit2, Gson Converter, Parcelor, ButterKnife, Glide, Firebase 등
다양한 Open Source까지 활용하였습니다.

Node.js

Pet Advisor의 Server는 Node.js를 통해 구축하였습니다. 기본적으로 비동기 IO 처리 방식이기 때문에 성능이 빠른 것이 장점입니다.
Express라는 프레임 워크와 body-parser 미들웨어를 활용하였고 multer, file system 등의 모듈을 활용하였습니다.

MariaDB

Database Server로 오픈소스 DBMS인 MariaDB를 사용하였습니다. 오픈소스이기 때문에 누구나 무료로 사용할 수 있으며, MySQL 기반으로 개발한 RDBMS이기 때문에 MySQL과 사용법이 동일합니다.

Pet Advisor 기능 상세 설명

아래는 Pet Advisor App을 통해 사용자가 이용할 수 있는 기능입니다.
다양한 기능을 활용하여 편의성을 느껴보세요.

불필요한 회원가입은 그만
카카오 계정 또는 네이버 계정을 통해 회원가입 없이 로그인할 수 있습니다.

서울시 동물 병원에 대한 정보
서울 열린 데이터 광장의 공공데이터를 활용하여
지역별, 거리순으로 볼 수 있으며 다양한 정보를 제공합니다.

카카오 맵을 통한 길찾기
각 병원의 주소 정보를 기반으로 병원까지의 경로를
카카오 맵을 통해 대중교통, 자동차, 도보 3가지 방식으로 알려줍니다.

병원 리뷰
리뷰와 함께 별점 평가와 즐겨찾기 또는 블랙리스트로 설정할 수 있습니다.
병원 방문자들의 진료 금액을 통해 방문 전에 진료 금액도 알 수 있답니다.
또한, 많은 사람들이 블랙리스트로 추가하면 사용자가 쉽게 파악이 가능합니다.

반려동물 관리
자신의 반려동물을 사진과 함께 등록해보세요.
사진은 Firebase 및 서버에 저장되며, 필요할 시 백업도 가능합니다.

반려동물의 이상 증상에 대한 검색
네이버 검색 API를 활용하여 블로그, 카페, 지식iN 에서
반려동물의 이상 증상에 대해 쉽게 검색이 가능합니다.
상세한 내용이 궁금한 경우 해당 게시글로 이동할 수 있습니다.

진료 내역 작성
반려동물마다 진료 내역을 기록할 수 있습니다.
예방 접종 날짜와 같은 중요 이벤트가 있다면 이벤트 날짜를 설정하세요.
Pet Advisor가 알림을 통해 사용자에게 알려줍니다.



Technique & Trouble Shotting

In Android

ViewPager와 Fragment 사용이 익숙치 않아 웹 서버에서 받아오는 데이터를 기반으로 화면 갱신이 제대로 이루어지지 않았다.
-> ViewPager의 setOffscreenPageLimit 메소드를 통해 모든 프래그먼트를 처음에 생성 및 Adapter를 새로 바꾸어줌.

LoginActivity에서 MainActivity로 화면 전환을 위해 startActivity를 호출해 주는데 MainActivity가 여러번 실행됨.
-> Manifest의 launchMode = "singleTop", taskAffinity="" 속성을 부여하여 해결.

RecyclerView의 아이템중에서 몇 개만 보여주고 다른 아이템은 visibility를 gone으로 설정 했는데 공간을 차지.
-> 화면에 보여줄 List를 새로 생성한 후 버튼 클릭 이벤트마다 아이템 추가 후 화면 갱신을 통해 해결.

In Node.js(웹서버)

로컬 서버 구축 시 커넥션이 끊기는 등, 통신이 원활하지 않음.
-> 고정 ip를 갖는 서버를 구축하여 해결.

안드로이드로부터의 파일 수신을 위해 multer 모듈을 사용하는데 req.field 방식으로 값을 가져오지 못함. -> req["field"] 방식으로 값을 참조할 수 있었음.

MariaDB 서버에 imagePath 전송하는데 imagePath에 \가 포함 되어 있어 정상적으로 저장되지 않음. -> replace 메소드를 통해 \\를 \\\\로 바꾸어줌.

In MariaDB

MariaDB는 RDBMS이기 때문에 동일한 이름을 갖는 칼럼을 여러개 가질 수 없음.
-> 칼럼 명과 동일한 새로운 테이블을 생성 후에 클라이언트가 요청하면 조인을 통해 데이터를 전송해줌.

List 내에 List가 있어 클라이언트가 List 사이즈만큼 루프를 돌면서 서버에 계속 요청해 성능이 떨어짐.
-> 테이블 간 조인과 MySQL 함수인 GROUP_CONCAT을 통해 한 번의 요청으로 모든 데이터를 받아올 수 있게 됨.

후기

현재까지 완성된 앱은 UI도 미흡하고, 기능적인 측면도 아쉬운 것 같습니다.
App Store에 등록할 때에는 UI도 다듬고, 병원 검색 기능과 실제 병원 데이터들을 추가할 예정입니다.
지금까지 해왔던 개발보다 부스트캠프에서 했던 것이 더 많았던 것 같습니다.
작년 봄까지만 해도 this, static 같은 자바 기초 지식도 몰랐던 제가 부스트캠프를 통해 주니어 개발자로 성장할 수 있는 발걸음을 내딛을 수 있었습니다. 안드로이드 개발 경험이 거의 없어서 처음엔 걱정도 많았지만 튜터님들과 조원들의 도움덕에 개발팁, 개발에 필요한 지식들을 쌓아 성장할 수 있었습니다.
오픈 소스를 뜯어보고 활용하는 모습에 저 스스로도 놀랐고 제가 어떤 새로운 모습을 보여줄 수 있을지 기대할 수 있게 된 부스트캠프 활동이었던 것 같습니다.
